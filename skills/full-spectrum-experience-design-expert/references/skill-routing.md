# 辅助 Skill 路由

## 使用原则

每一步只选择能实质改善输出的最小 Skill 集。选择后必须读取对应 `SKILL.md` 并遵循其完整流程。若本机没有该 Skill，使用最接近的方法继续，不得阻断主流程。

| 步骤/问题 | 优先辅助 Skill | 用途 |
|---|---|---|
| PPT读取、模板回填、导出与QA | `Presentations` | 模板审计、artifact-tool编辑、渲染和一致性检查 |
| 原始卡点多、描述杂乱 | `亲和图分析` | 自下而上聚类、洞察和证据强度 |
| 挑战优先级与路线图 | `机会框架` | 影响/投入、RICE、Kano或价值/复杂度 |
| 三条体验原则 | `设计原则` | 立场、取舍、可测试性和优先级 |
| 功能多、对象关系复杂 | `信息架构` | 内容模型、对象关系、导航和可发现性 |
| 多设备/大数据检索 | `搜索体验` | 搜索入口、分面筛选、排序、零结果和搜索分析 |
| 安装、购买、激活流程 | `表单设计`、`用户流程图` | 减少步骤、预防错误、明确分支和恢复 |
| 弱网、上传、回放、权限错误 | `错误处理体验`、`反馈模式`、`状态机交互` | 预防、检测、沟通、恢复和状态可见性 |
| JTBD任务定义 | `待办任务分析` | 功能/情感/社会任务与结果期望 |
| 全链路阶段与情绪 | `用户旅程地图` | 阶段、触点、情绪、断点、机会和关键时刻 |
| 前台/后台/支撑协同 | `服务蓝图` | 系统依赖、SLA、交接和失败点 |
| 报告、看板和价值证明 | `数据可视化` | 图表选择、目标对比、异常解释和可行动洞察 |
| 指标体系、目标与追踪 | `指标定义` | HEART/任务成功/行为/商业指标与Owner |
| 通知、加载、微交互 | `加载状态`、`微交互规格`、`多尔蒂阈值` | 反馈时机、响应阈值和渐进呈现 |
| 隐私、权限与可访问性 | `可访问性审计`、`错误处理体验` | 权限解释、审计、敏感操作和可访问反馈 |

## 组合建议

- 智能车队/IoT 平台：亲和图分析 + 搜索体验 + 信息架构 + 错误处理体验 + 指标定义 + 服务蓝图。
- 消费级智能硬件：设计原则 + 待办任务分析 + 用户旅程地图 + 错误处理体验 + Hook分析。
- 数据看板产品：信息架构 + 搜索体验 + 数据可视化 + 指标定义。
- 多角色企业产品：信息架构 + 状态机交互 + 错误处理体验 + 服务蓝图。

辅助 Skill 不能改变主工作流顺序；其产出必须写回当前 Step 并向后传递。
